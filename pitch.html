01 · The Problem

The Tyranny of Hierarchy

For fifty years we've lived inside the same information structure. A page you scroll top to bottom. Files nested in folders. Hyperlinks jumping between documents. Whether it's your local filesystem, a website, or a library — the skeleton is identical: a tree.

📚

The Library Metaphor

Think of a physical library. Books (files) sit on shelves (folders). You walk between sections. You open a book and scroll page by page (1D). The internet is the same library — websites are buildings, links are corridors. We never left the metaphor.

Trees force one organizational perspective. A file can only live in one folder. A book sits on one shelf. But knowledge is intertwingled — every concept connects to every other in ways that hierarchies cannot express.

The first breakthrough is simple but radical: remove the tree. Every file becomes a media container — a card that can hold source code, images, documents, interactive components — placed on an infinite canvas.

  • Source code files with syntax highlighting & inline diffs
  • Images & diagrams at full resolution
  • Interactive components — embeds, charts, live previews
  • Markdown documents with rendered formatting

You can move them anywhere. Arrange them by topic, by urgency, by team, by whatever pattern makes sense to you.

🗺️

The Minimap Context

An infinite canvas without orientation is chaos. The minimap solves this. It's a bird's-eye view showing relative positions. Click to jump. It gives you back the context that the tree used to provide, without the constraints.

03 · Layers

The Fourth Dimension: Perspectives

🍎

Color or Taste?

Do you organize fruits by color or by taste? In a hierarchy, you must choose one. The apple goes in "red" or "sweet" — never both. A file goes in src/auth/ or src/api/ — never both. Every hierarchy is a single, frozen perspective.

Layers are our answer. Each layer is a different projection of the same underlying files. The files don't move — they exist once. But each layer defines a different spatial arrangement. Think of it like a prism splitting white light:

  • 🏗️ Architecture Layer: Grouped by module boundaries.
  • 🔗 Dependency Layer: Arranged by import chains.
  • 🔥 Churn Layer: Sized dynamically by change frequency.
  • 👥 Team Layer: Code ownership becomes spatial territory.

You're no longer asking "where does this file live?" but "how does this relate from this perspective?"

04 · Time Navigation

The Fifth Dimension: Filmstrip

- function flatWorld() {
+ function spatialEngine() {

Knowledge evolves. Code changes. Geopolitics shifts. The fifth dimension lets you navigate through time — scrubbing backward and forward through snapshots of your canvas.

In GitMaps: Every Git commit is a frame. Use arrow keys to step through history. The canvas updates — files appear, disappear, change. Inline diffs show exactly what changed. You're watching your codebase evolve as a film.

In WARMAPS: Instead of Git commits, WARMAPS uses interval ticks (minutes, hours). Scrub the timeline to see the state of any monitored region in the past. Troop movements, flight paths, news headlines — all rewound or fast-forwarded.

🎬

The Security Camera

Think of a security camera system. Each camera is a container. The timeline is a VCR scrubber. You can review any moment, from any angle. That's what the fifth dimension gives every knowledge domain.

05 · Connections

Intertwingularity Made Visible

Ted Nelson's core idea was that knowledge is a web, not a tree. Our connections feature makes this literal.

Alt + click a line in one file, pick a target file, click a target line — and a permanent, visible link is drawn between them.

These connections persist across layers and time. They are the cross-references that hierarchies cannot express — the "see also" that exists in your head but never on your screen. Until now.
