# Git4D.space — 4-Dimensional Knowledge Management

## Core Concept: Dimensions of Reading

| Dimension | What | How |
|-----------|------|-----|
| **1D** | Linear reading | Traditional book/file/website — sequential text consumption |
| **2D** | Temporal reading | Git adds time axis — see how files transform across commits |
| **3D** | Spatial reading | Canvas transclusion — all files projected into shared visual space |
| **4D** | Layered reading | Multiple layer projections — different perspectives on same codebase |

## Architecture

- **1D → 2D**: Git history / diff visualization (already solved by git)
- **2D → 3D**: Magic of transclusion — every file in repository rendered as a card on infinite canvas
- **3D → 4D**: Layer system — project files into categorized layers (by function, by author, by date, by AI-analysis)
