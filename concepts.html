The Five Dimensions of Knowledge

A deep dive into the universal formula behind GitMaps. How we transcend hierarchies and give every knowledge domain — from code to geopolitics — five degrees of navigational freedom.

01 · The Problem

The Tyranny of Hierarchy

For fifty years we've lived inside the same information structure. A page you scroll top to bottom. Files nested in folders. Hyperlinks jumping between documents. Whether it's your local filesystem, a website, or a library — the skeleton is identical: a tree.

📚

Think of a physical library. Books (files) sit on shelves (folders). You walk between sections (navigate the hierarchy). You open a book and scroll page by page (1D). The internet is the same library — websites are buildings, links are corridors between rooms, pages are pages. We never left the metaphor.

This works, but it's limited. Trees force one organisational perspective. A file can only live in one folder. A book sits on one shelf. But knowledge is intertwingled — every concept connects to every other in ways that hierarchies cannot express.

" Intertwingularity is not generally acknowledged — people keep pretending they can make things deeply hierarchical, categorizable and sequential when they can't. — Ted Nelson, Computer Lib/Dream Machines, 1974

These three dimensions — scroll, lateral navigation between files, vertical navigation through the folder hierarchy — are the default. The same for local filesystems, the web, and physical libraries. Both are derived from the physical world. We start here and break through.

02 · The Canvas

Breaking Free: Infinite Spatial Freedom

The first breakthrough is simple but radical: remove the tree. Every file becomes a media container — a card that can hold source code, images, documents, interactive components — placed on an infinite canvas.

You can move them anywhere. Arrange them by topic, by urgency, by team, by whatever pattern makes sense to you. There is no enforced structure — unless you choose to enter a read-only view where positions are locked for a shared perspective.

The Minimap

An infinite canvas without orientation is chaos. The minimap solves this. It's a bird's-eye view showing the relative position of every container. Click to jump, see the big picture in the corner, interact at the detail level — just like scrolling a file, but in two dimensions. The minimap gives you back the context that the tree used to provide, without the constraints.

Files as spatial containers — the minimap preserves global context

Media Containers

Each card on the canvas is a media container. It can render:

  • Source code files with syntax highlighting and inline diffs
  • Images, diagrams, and visual assets at any resolution
  • Interactive components — embeds, charts, live previews
  • Markdown documents with rendered formatting

Containers can be resized, collapsed, or expanded. Each user can arrange and resize them as they wish, and in collaborative mode, changes propagate in real time.

03 · Layers — The Key Breakthrough

The Fourth Dimension: Multiple Perspectives

🍎

Do you organize fruits by color or by taste? In a hierarchy, you must choose one. The apple goes in "red" or "sweet" — never both. A file goes in src/auth/ or src/api/ — never both. Every hierarchy is a single, frozen perspective.

Layers are our answer. Each layer is a different projection of the same underlying files. The files don't move — they exist once. But each layer defines a different spatial arrangement, a different way the pieces relate to each other.

Think of it like a prism splitting white light. One codebase. Multiple angles:

🏗️ Architecture Layer

Files grouped by module boundaries, showing how subsystems isolate and connect.

🔗 Dependency Layer

Files arranged by import chains — who depends on whom, what's tightly coupled.

🔥 Churn Layer

Files sized by how often they change. Hotspots become obvious at a glance.

👥 Team Layer

Files grouped by who owns them — code ownership becomes spatial territory.

Why this is a breakthrough

Hierarchies enforce a single organisational axis. Layers give you N axes simultaneously. Switch between them instantly. The same file appears in every layer — only its spatial context changes. You're no longer asking "where does this file live?" but "how does this file relate from this perspective?"

Layers in WARMAPS

In WARMAPS, layers are not code perspectives — they're geopolitical contexts. Each war, each country is a different layer. Instead of scrolling into a finite globe to find Ukraine, you select the Ukraine layer and see all its information components laid out on the infinite canvas: the map (with plane flights, troop positions) is just one container. Next to it are news feeds from Reuters, Al Jazeera, Telegram channels — each a resizable container. The user can reorganize them all, and it updates in real-time.

The key insight: a globe is a finite space. You're constrained. An infinite canvas with layers is unbounded. Each layer is a complete, spatial intelligence dashboard.

04 · Time Navigation

The Fifth Dimension: History as a Filmstrip

Knowledge evolves. Code changes. Geopolitics shifts. The fifth dimension lets you navigate through time — scrubbing backward and forward through snapshots of your canvas.

In GitMaps: Git Commits

Every Git commit is a frame. Use ← → arrow keys to step through history. The canvas updates — files appear, disappear, change. Inline diffs show exactly what changed between frames. You're watching your codebase evolve as a film.

Inline diffs — see exactly what changed between time frames

In WARMAPS: Timeline Ticks

Instead of Git commits, WARMAPS uses interval ticks — minutes, hours, days. Scrub the timeline to see the state of any monitored region at any point in the past. Troop movements, flight paths, news headlines — all rewound or fast-forwarded.

🎬

Think of a security camera system. Each camera is a container. The timeline is a VCR scrubber. You can review any moment, from any angle. That's what the fifth dimension gives every knowledge domain.

05 · Connections

The Intertwingularity Made Visible

Ted Nelson's core idea was that knowledge is a web, not a tree. Our connections feature makes this literal. Alt+click a line in one file, pick a target file, click a target line — and a permanent, visible link is drawn between them.

Cross-connections between files — the relationships your file tree hides

These connections persist across layers and time. They are the cross-references that hierarchies cannot express — the "see also" that exists in your head but never on your screen. Until now.

06 · The Engine

GalaxyDraw: The Universal Renderer

Behind both GitMaps and WARMAPS is a single rendering engine: GalaxyDraw (internally "galaxy-canvas"). It provides:

  • Infinite canvas with hardware-accelerated pan/zoom
  • Minimap with real-time container position tracking
  • Media containers — pluggable renderers for any content type
  • Layer system — N projections of the same data set
  • Timeline — frame-by-frame navigation with pluggable time sources
  • Connection graph — line-level cross-references between containers
  • Grid & snap — optional structural enforcement for shared views

The engine is domain-agnostic. GitMaps feeds it files, diffs, and Git history. WARMAPS feeds it maps, news, and OSINT data. Any knowledge domain can be plugged in — research papers, legal documents, medical records, design systems.

The Universal Formula

Canvas + Containers + Minimap + Layers + Timeline + Connections = Five-dimensional knowledge exploration. This is the formula. GalaxyDraw implements it. GitMaps and WARMAPS are merely the first two instantiations.

07 · Use Cases

GitMaps & WARMAPS

🗺️ GitMaps

Spatial code exploration for developers.

  • Containers = source files, images, configs
  • Layers = architectural perspectives
  • Time = Git commit history
  • Connections = cross-file references
  • Minimap = codebase-at-a-glance

⚔️ WARMAPS

Real-time geopolitical intelligence dashboard.

  • Containers = maps, news feeds, flight data
  • Layers = countries, wars, conflict contexts
  • Time = interval ticks (minutes/hours)
  • Connections = event correlations
  • Canvas = unbounded (not a finite globe)

Why Not Just a Map?

A traditional map application gives you a finite globe. You scroll to a country, you zoom in. You're constrained by geography. WARMAPS breaks this by using the infinite canvas model: instead of scrolling to Ukraine on a globe, you select the Ukraine layer. Now you see all its components — the geographic map is just one container. News feeds from different sources are other containers. Flight data is another. Telegram OSINT channels are another. Each user can resize and reorganize them, and everything updates in real-time.
