The file system lied to you.

We organize code into directories and files because that's what computers demand. But knowledge doesn't live in trees. Ideas connect sideways — across files, across modules, across the neat little folders we construct.

A function in auth.ts is deeply connected to a validation rule in schema.ts, which shapes a form in signup.tsx, which triggers an email defined in mailer.ts. These relationships are the actual architecture — but your editor shows you one file at a time.

Git on Canvas breaks the file hierarchy open. Every file becomes a card on an infinite canvas. You draw connections between lines. You see the web that was always there — the intertwingularity your file system was hiding.

Code is knowledge.
Knowledge is a web.

Since the 1960s, Ted Nelson imagined a world where information is linked — not filed. Where every idea connects to every related idea, and the connections themselves carry meaning.

We inherited the file system from an era of punch cards and disk platters. But your code's actual structure — the one in your head — is a graph of relationships, not a tree of directories.

Git on Canvas is a step toward Nelson's vision. Not a full hypertext system — just a practical tool that lets you see and draw the connections that your IDE flattens into a tab bar.

" There are no 'subjects' at all; there is only all knowledge, since the cross-connections among the myriad topics of this world simply cannot be divided up neatly. — Ted Nelson
